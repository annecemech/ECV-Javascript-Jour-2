1. Interagir avec un bouton

Au clic sur le bouton, changer le contenu du texte "cliquez ici" par "Bravo!"

2. Rendre une FAQ dynamique

Voici une FAQ qui doit être sous forme d'accordéon. Lorsque l'on clique sur un élément, tous les autres doivent se replier et seul celui sur lequel on a cliqué doit être visible.

Sélectionner tous les éléments qui portent la classe .accordion-item avec document.querySelectorAll et stocker les dans une variable (const).

Itérez sur chacun des éléments (forEach) et pour chacun de ces élément, posez un écouteur (addEventListener). Au clic, ajoutez ou retirer la class .active (classList.toggle).

Attention pour sélectionner l'élément .accordion-item qui a été cliqué, faut utiliser "event.currentTarget" ou "event.target" ?

2. Faire des suggestions de recherche

Le but de l'exercice est de détecter ce que l’utilisateur tape dans l’input, pour lui afficher une suggestion de villes correspondantes dans une liste <ul> qui porte l'id #results (inspectez le code pour la trouver). La liste doit être mise à jour en fonction de la recherche, et ne doit pas accumuler les résultats indéfiniment.

La liste des villes se trouve déjà dans le fichier script.js dans une constante cities.

Sélectionnez l’élément <input> avec document.getElementById et stockez-le dans une variable (const).

Posez un écouteur sur l’input avec addEventListener sur l’événement "keyup" pour détecter chaque saisie.

Récupérez la valeur de l’input avec .value et utilisez filter() pour chercher dans le tableau cities les villes qui incluent ce texte.

Affichez les résultats dans la liste <ul> en créant dynamiquement des <li>. Pensez à utiliser insertAdjacentHTML pour mettre à jour la liste.

Pour insérer dynamiquement le contenu dans le HTML, utilisez l’interpolation de chaînes de caractères avec les template literals :
Exemple : `<li>${nomDeLaVille}</li>`.
Consultez la documentation sur les template literals pour en savoir plus.

Bonus 1 : Avant d’ajouter de nouveaux résultats, videz la liste pour éviter l’accumulation des éléments.
Astuce : Assurez-vous que la liste est bien mise à jour en réassignant innerHTML = "" avant d’insérer les nouvelles suggestions.

Bonus 2 : Rendez la recherche insensible à la casse.
Pensez à convertir à la fois la valeur de l’input et les villes de cities en minuscules avec toLowerCase() avant d’effectuer la comparaison dans filter().
